>
> 整合自 2024–2026 年顶尖研究者、创业者、工程师的深度访谈与公开写作，截至 2026-07。不按人物罗列，而按这些人实际在回答的根本问题重组：AI 的本质是什么、能力边界由什么决定、瓶颈往哪里迁移、护城河在模型还是 harness、AGI 有多远、人和组织该怎么变。每个问题下先给收敛的判断，再给真正的分歧。

## 问题一：AI 到底是什么

关于 AI 本质的比喻不是文字游戏——你怎么理解它，决定了你怎么用它、怎么预判它的失败方式。几个最值得听的人给出了不同侧面的答案，有意思的是，这些比喻彼此并不冲突，反而拼出了同一幅画。

**Karpathy 的"幽灵"**（Dwarkesh 播客，2025 年 10 月）：AI 是模仿人类数据训练出的幽灵（ghost），不是进化出来的动物。没有内在动机、没有具身常识，是带 RL 附属物的统计模拟电路。关键推论是：前沿模型的能力"参差不齐"（jagged），因为实验室只为自己在意的任务搭 RL 环境——能力轮廓不是自然生长的，是被人工雕刻的。

**Willison 的"过度自信的实习生"**：一个读过世界上所有教科书、但对你的特定代码库零实战经验、且记不住一小时之前任何事的初级开发者。"Over-confident"这个定语不能省——它提醒你模型不会说"我不确定"。他的另一面更冷："忽略 AGI 炒作，LLM 仍是花哨的自动补全"——但写代码恰恰主要就是按正确顺序串 token，所以只要指对方向就极其有用；假设它能在你不动用任何技能的情况下完美完成项目，你会很快失望。

**Howard 的"插值器"**：LLM 擅长在已知之间填空，弱于真正的分布外概念创新。2025 年的 agent 脚手架之所以 work，是把更多问题"挪进分布内"，而不是修复了基础模型的理解短板。

**Mollick 的"锯齿边界"（jagged frontier）**：AI 的能力不是一条均匀的前沿线，某些任务它超乎预期地强，某些任务出人意料地弱，而且不用就不知道边界在哪。这个概念在 2026 年 3 月有了正式实证：Dell'Acqua 为第一作者、Mollick 为共同作者的 BCG 顾问预注册实验（758 人）发表于 Organization Science——用 GPT-4 的顾问多完成 12.2% 的任务、平均快 25.1%、质量显著更高；但在"边界之外"的任务上，正确率比对照组低约 19 个百分点。AI 不是均匀增益，它重新分配了谁在哪里犯错。

**Lilian Weng 的幻觉拆解**：幻觉分两种——输出该忠于上下文却没有（in-context），和输出该被世界知识支撑却没有（extrinsic）。根治需要两个能力：事实准确，以及不知道时明确说不知道。当前模型两项都不够好——这正是"过度自信的实习生"的技术注脚。

把这些拼起来：**AI 是一个能力轮廓被训练信号人工雕刻出来的、不会承认无知的强大插值器**。四个比喻各自强调了一个失败模式：幽灵提醒你别赋予它动机，实习生提醒你必须 review，插值器提醒你别指望分布外创新，锯齿边界提醒你逐个任务实测。这是本文所有后续判断的地基。

## 问题二：能力从哪里来，边界由什么决定

这是全部素材中收敛最强的一个判断，三条独立证据链指向同一个答案：**AI 能做好什么，由"是否存在清晰的验证信号"决定，而不是由任务难度决定。**

**第一条链，Karpathy 的框架**（Sequoia Ascent 2026）：传统软件自动化你能"指定"的东西；LLM 和 RL 自动化你能"验证"的东西。这解释了为什么 math / coding / tests 进步最快——代码能不能跑、测试过不过，信号清晰。推论：要让 AI 做得好，你得先想清楚"什么叫做好"；没有验证标准的任务，AI 只能靠一次猜对。

**第二条链，姚顺宇的 95% 论**（张小珺访谈 #140，2026 年）：据其自述，作为参与过 Claude 3.7/4.5、Gemini 3 关键开发的研究者，他的判断是"95% 的白领工作其实比写代码简单，但大模型做不掉的关键在于没有一个好的信号来评判好坏"。评判标准缺失导致 RL 训不出来。Coding 先突破不是因为它简单，而是因为它反馈信号最清晰。

**第三条链，竞赛与部署数据**：ARC 2024 的三个顶级方案（ARChitects、Akyürek 团队、MindsAI）独立收敛到 test-time training——在有明确对错信号的任务上，测试时的针对性适应就能大幅提升表现。（ARC 2025 论文奖的 Tiny Recursive Model 以 7M 参数达到 ARC-AGI-1 的 44.6%，常被归入同一证据，但它是在任务数据上从头训练的小模型，方法归类存疑，不宜直接当作 test-time training 的证据。）

三条链来自不同位置——框架提炼、一线训模型者的经验、竞赛结果（同处一个行业话语圈，独立性并不完全，但位置差异足以互证）——指向同一结论，这是素材里强度最高的共识。它同时解释了两件表面无关的事：为什么 coding agent 一骑绝尘，以及为什么就业冲击最先落在初级白领岗位（见问题六）。

与它配套的第二层共识是**范式重心的迁移**，四个完全不同位置的人在两年内说了同一件事：

- Chollet（AI Startup School，2025 年 6 月）从研究侧宣布："2025 年我们已经彻底告别预训练 scaling 范式，进入 test-time adaptation 时代"——模型在推理时根据具体数据动态调整行为，涵盖 test-time training、program synthesis、chain-of-thought synthesis，"如今每一个在 ARC 上表现好的方法都在用其中之一"。
- 罗福莉（张小珺访谈 #138，2026 年 3 月）从产业侧："大模型战争掀开第二幕——从 Pre-train 主导的 Chat 时代，转向 Post-train 主导的 Agent 时代。上一个时代的成功不意味着下一个时代的领先，现在基本上大家在同一水平线。"
- Jensen Huang 从硬件侧："Inference is thinking，思考远比阅读难，预训练只是记忆。"他公开阐述过三条 scaling law：预训练、后训练、测试时（另有"第四条 agentic"的版本流传，出处待核实）。Hassabis 的量化观察支持同一方向：同一模型关掉思考"大概是大师级"，打开思考"远超世界冠军级，相差 600+ ELO"——test-time compute 带来的是质变。
- 姚顺宇从训练一线补了打通两端的一句："Pre-train 没有到头——Pre-train 也是一种 RL。"即便预训练本身，也在向强化学习的逻辑收敛。

重心从"训练时学到的"移向"推理时算出来的、信号里训出来的"，这是 2025–2026 最少争议的行业判断。

## 问题三：瓶颈在哪里，往哪里迁移

多个来源不约而同在讲同一种结构：**AI 消灭一个瓶颈，瓶颈就迁移到旁边那个更难量化的环节**。至少四次迁移已经发生或正在发生。

**从写 prompt 到管 context。** Karpathy（2025 年 6 月）："在每个工业级 LLM 应用里，上下文工程才是用恰当信息填充上下文窗口的精妙艺术与科学。" Riley Goodside 早在 2023 年初就区分过 context engineering（为任务挑选和准备相关上下文）与 prompt programming（写清晰指令）；2025 年 11 月他给出了更锋利的表述："提示词工程是操心用户该写什么，上下文工程是操心模型该读什么——两者过去是同一回事，少有人预见它们的分化。" Karpathy 的 Software 3.0 框架把这推到哲学层面：1.0 是代码，2.0 是学习到的权重，3.0 是把提示当编程——上下文窗口是杠杆，LLM 是解释器。

**从写代码到代码周围的一切。** Fiona Fung（Code with Claude 2026 London）的总结可能是 2026 年最值得记住的一句话："瓶颈从写代码转移到了写代码周围的一切。"写 PR 的速度不再是限制，新瓶颈是 review、验证、协调、架构判断、决定做什么。她的应对办法本身就是新范式的样本——"技术辩论中，代码赢"：让 Claude 把两个方案都原型化出来再比，而不是空谈设计文档。Karpathy 在 Sequoia Ascent 2026 给了这个迁移一个分层：vibe coding 抬高地板（任何人都能造软件），agentic engineering 抬高天花板（人仍对质量负责，瓶颈变成品味和编排技能）。他举的例子很具体：让 Claude 做 Stripe-Google 账户匹配，代码能跑，但设计错了——这是品味缺失，不是技术缺失。

**从算力到能源。** Zuckerberg（Dwarkesh，2024）：有钱可以买 GPU，但没法用钱凭空变出能源和电力审批。这与行业普遍盯着"GPU 供应"的视角不同。Jensen Huang 则反驳"推理芯片会变小变便宜"的共识：推理是思考、是把新问题分解成可解小块，怎么可能计算轻量？他预测用户数指数增长乘以单次计算量指数增长的双指数——被问及一年前的预测时坦承"我真的低估了"。

**从技术到制度。** Mollick（2026）：AI 让药物发现快 10 倍，但瓶颈变成 FDA 审批速度——"制度以制度的速度移动"。AI 加速的不只是技术，它把约束推到了制度层面。

这四次迁移有一个共同方向：**瓶颈总是从"可自动化的"移向"需要判断的"**——从生成移向验证，从执行移向决策，从技术移向治理。这与问题二的验证信号框架是同一枚硬币的两面：AI 吃掉所有信号清晰的环节，剩下的自然全是信号模糊的。

## 问题四：护城河在模型，还是在模型之外

这是素材里第一个真分歧所在，但先说数据，因为数据本身出人意料。

**Scaffold 对表现的影响已经和模型本身同量级。** SWE-bench 2026 年的数据（下列具体数字均转述自行业讨论，未经一手来源核验，取其量级与现象即可）：六个前沿模型彼此相差不到 0.8 分，但同一个模型在不同 agent 系统下差 5.2 分。同一个"SWE-bench Pro 最好成绩"有至少三个数字同时流通（标准化 scaffold 下 59.1%、厂商自家 scaffold 下 69.2%、商业闭源子集下 47.1%〔待核实〕）——引用任何分数都必须说明条件。更糟的是基准与产品的脱节：Augment 报告"改善生产 agent 的改动往往不动 SWE-bench 分数"，METR 发现许多通过 SWE-bench 的 PR 过不了人工 review、模型在长任务上急剧退化。SWE-bench Verified 本身据称在 2026 年被 OpenAI 公开放弃（审计发现 59.4% 最难未解题有缺陷测试用例；此说法与数字待核实）。结论：把基准分数当能力上界而非真实工程能力，重要选型用任务特定 eval 收尾。

开发者社区的经验与此呼应：技术社区偏好 Claude API 的核心原因不是能力更强，而是指令遵从更稳定。Reddit r/MachineLearning 上的高票观察：指定"用 React hooks，不要 class components"，Claude 遵从；GPT-4o 经常回复"你也可以考虑……"然后无视要求。有团队报告切换后，"AI 做了奇怪的事"类 support ticket 减少了 70%。认知含义比表面深：**工程场景里，可预测性和可控性比偶尔的超水平发挥更有价值**——一个 90 分但稳定遵从指令的模型，在生产环境中往往优于一个 95 分但时不时自作主张的模型。

在这些数据之上，戴雨森（真格基金，张小珺访谈 #142，2026 年 5 月）给出了最有解释力的结构：**"Harness 更像 OS，模型更像处理器"**——模型决定性能上限，harness 决定你实际能发挥多少；栈结构是 model → harness → context → runtime。

推论有两层。其一，好的 harness 有生命力、难以被替换，好的模型却容易被下一代取代；真正的护城河不是用了哪个模型，而是积累的 harness 数据——它能反哺训练形成数据飞轮，而普通聊天对话数据带不来新知识。其二，harness 会腐化：它对模型能力的假设随模型迭代变旧，常见症状是过时的 few-shot 样板、为旧模型设计的防御性约束、基于旧模型弱点的 workaround——定期审计 harness 假设是否仍成立，是维护其价值的必要成本。

**但这个判断有真正的对手方。** 李广密（拾象，张小珺访谈 #136，2026 年 5 月）的判断方向相反：模型正在成为下一代 OS（新的计算平台），Coding 是这个 OS 最核心的入口，也是 AI 自我加速的引擎——领先的 coding 模型给顶尖人才 10–50x 杠杆，类比"领先 GPU 给算力带来的加速"。他认为 Opus 4.5→4.6 是 coding agent 从"还不够好"跨到"真的可以用"的拐点（类比 GPT-3→GPT-4）。

Freda Duan（Altimeter，张小珺访谈 #141）的飞轮论支持模型侧："better AI makes better AI"——AI 能力提升让 AI 研究本身更便宜、模型迭代更快、能力继续提升，飞轮一旦转起来，追赶者的时间窗口越来越短。她还提出 token 是误导性计量单位：同一任务在不同模型上 token 消耗差 10–100 倍，按 token 收费实际上在惩罚高效率的模型，行业正在转向按效果收费（outcome-based pricing）——这同样把价值锚定在模型效率而非外围工程上。

而梁文锋（36氪暗涌访谈）从第三个角度削弱"模型护城河"论："在颠覆性技术面前，闭源形成的护城河是短暂的，即使 OpenAI 闭源也无法阻止被赶超。开源更像一个文化行为，而非商业行为。"如果最强模型的领先优势注定短暂，那么模型侧的飞轮论就要打折扣。

**这个分歧怎么裁决？** 部分可调和：戴雨森和李广密说的"OS"不在同一层——李广密说的是模型作为计算平台的产业地位，戴雨森说的是应用层内部 harness 与模型的分工，两者可以同时为真。真正对立的是投资含义：护城河沉淀在模型厂商，还是沉淀在拥有 harness 数据和工作流的应用方。

判别信号有两个。一看换模型时应用方的迁移成本走势——MCP 这类标准化合约正在压低它。社区和 MCP 官方反复纠偏：MCP 不是"给 AI 加工具"的便利层，而是 AI 与外部世界之间的标准化合约，类比 USB-C，切换底层模型不需要重写任何工具集成（另一个常混淆的区分：MCP 管 agent 连工具和数据源，A2A 管 agent 连 agent，两者互补不替代）。迁移成本越低，模型越商品化，harness 侧越占优。二看新一代模型发布时，上一代 harness 上积累的优势是被继承还是被清零——如果每代模型都把 harness 优势清零，李广密对；如果 harness 数据持续复利，戴雨森对。目前 SWE-bench 的 scaffold 数据（同模型差 5.2 分）短期更支持 harness 侧，但 Freda 的飞轮论在长期没有被反驳。

## 问题五：AGI 有多远——光谱、真分歧与系统性偏差

先摆光谱，再挖分歧的本质。

**保守端。** Chollet 和 LeCun 的立场：纯 scaling 到不了 AGI。这不只是两个人的直觉——AAAI 2025 年调查 475 名社区受访者（约 67% 来自学界），76% 认为"放大现有方法"产生 AGI"不太可能"或"非常不太可能"。Chollet 的理由是定义性的："人类考试不是为测量智能设计的"——智能是高效获取新技能、解决未曾训练过的新问题的效率比率，不是考高分；他偏好深度学习加 program synthesis 的路径。Hassabis 温和一些："我猜测还需要一到两个大突破——Transformer 级别或 AlphaGo 级别的。"但他同时给了一个不该被忽略的量化观察：同一模型关掉思考"大概是大师级"，打开思考"远超世界冠军级，两个版本相差 600+ ELO"——test-time compute 带来的是质变而非渐进。

**激进端。** Altman（《The Gentle Singularity》，2025 年 6 月）：心智模型是"奇迹变成常规，再变成基本标配"，预测 2026 年很可能见证能找出新颖洞见的系统。Dario Amodei（《Machines of Loving Grace》，2024 年 10 月）：大多数人低估了 AI 上行空间有多激进——强 AI 是能设计实验、发明方法的"虚拟生物学家"，可把一个世纪的生物学进展压缩到 5–10 年。注意两人的诸多陈述是前瞻性预测，用未来/条件时态表达，不应当作已发生的事实。

**光谱在移动，且向中间收敛。** 2026 年 Chollet 把自己的 AGI 时间线从"不确定"缩短到约 5 年，并着手设计 ARC-AGI-3——约 100 个手工 agentic 交互式游戏环境，专门测量"尚未被攻破的 agentic 智能"。他不再认为 AGI 遥不可及，但仍坚持当前路径不够。另一边，Dario 在 2026 年 1 月的《The Adolescence of Technology》中明确转向风险讨论（目标错位、小团体滥用、强权者滥权、经济颠覆、未知效应五分法，同时反对末日论的准宗教式思维）。两端都在向"近，但有条件"靠拢。

**分歧的本质不是时间线，是"缺不缺新东西"。** Altman/Dario 的立场蕴含"当前路径 + 算力 + 工程就够"；Chollet/LeCun/Hassabis 认为还缺至少一个未发明的组件。这是真分歧，不是措辞差异，而且短期内不可调和——它只能被时间裁决。判别信号：① ARC-AGI-3 这类专测分布外 agentic 能力的基准，是被现有路径的下一代模型直接攻破，还是持续顶住；② Altman 预言的"能找出新颖洞见的系统"在 2026–2027 是否真的出现（注意用 Chollet 的标准检验：是分布外创新，还是把问题挪进了分布内）；③ Hassabis 说的"下一个大突破"如果出现，AGI 归功于它还是归功于 scaling。

**必须叠加一层系统性偏差校正。** 激进端全部是实验室领袖——Altman、Dario 的预测若成真，公司估值直接兑现；保守端以学界为主——AAAI 调查 67% 受访者来自学界，对 scaling 的成功没有敞口，甚至有反向的立场敞口（scaling 若真够用，大量学术研究议程贬值）。两边都有利益相关，任何一边的话都不能按面值全收。

历史记录提供一点校准参考。insider 侧最坦诚的一次来自 Jensen Huang：被问及一年前对推理计算需求的预测时承认"我真的低估了"——注意方向，insider 在能力和需求上反而常常不够激进。outsider 侧，Chollet 在数年内把时间线从"不确定"收缩到"约 5 年"，说明保守端一直在被证据推着走。这两个例证给出一个可参考的方向（样本有限，不是定律）：**insider 在"能力会到什么程度"上更常对，outsider 在"要花多长时间"上更常对**。读预测时按这个方向打折，比选边站更可靠。

**Karpathy 是光谱上最有意思的一个点，因为他看起来自相矛盾。** 一方面极度克制：他被行业"智能体元年"的说法激怒，坚持更准确的说法是"the decade of agents"，并借自动驾驶的经验讲"march of nines"——做出 90% 能用的 demo 只是第一个 9，之后每一个 9 都是等量的工作；AGI 会融入约 2% 的 GDP 增长，是过去 250 年自动化进程的延续而非断裂，认知核心可能小到 10 亿参数级别。另一方面又非常激进："自动化你能验证的"框架意味着在信号清晰的领域（coding）替代几乎没有上限，瓶颈只剩品味和编排。

这两面其实是同一个模型的两个轴：**在可验证域内，变化是断裂式的；在整体经济层面，变化是连续的**——因为经济由大量弱信号任务构成（姚顺宇的 95%），锯齿边界的宏观表现恰恰就是"局部革命、总量渐进"。读 Karpathy 不应二选一，他的克制和激进各自适用于不同的轴；这也是理解本文附一就业数据"两面并存"的钥匙。

## 问题六：人和组织该怎么变

### 组织形态：自下而上与自上而下都赢了，这本身是个问题

素材里有一组真实的对照。自下而上的一边：DeepSeek（梁文锋）："并没有什么高深莫测的奇才，都是 top 高校的应届毕业生、没毕业的博士、博五实习生。选择标准一直是热爱和好奇心。全是自下而上，一般不前置分工，而是自然分工。"MiMo（罗福莉）：约 100 人团队"没有职级、没有小组划分、没有 deadline"，训 1T 模型核心团队只有几个人——"平权本身是有价值的，任何层级一定程度都是规范和约束，而规范和约束本身是压制创造力的"。

自上而下的一边：Anthropic（据姚顺宇自述及虎嗅报道）是自上而下、文化面试一票否决的组织。姚顺宇据其自述，离开的原因约四成是与 Anthropic 把中国列为"敌对国家"的价值观分歧——这也提示自上而下组织的一种代价：价值观一致性筛选在凝聚文化的同时，也在按非技术维度流失人才。

关键在于：**两种组织都产出了前沿模型**。所以这不是"哪种组织形态对"的问题，而是环境依赖：追赶和探索期（DeepSeek、MiMo 的处境）自下而上的方差是资产；守成和对齐敏感期（Anthropic 的处境）自上而下的一致性是资产。真正值得记住的信号是罗福莉那句"上一个时代的成功不意味着下一个时代的领先"——范式切换期，组织的重构速度可能比组织形态本身更重要。

### 个人：靠谱、品味、领域知识——三个都反"天才叙事"

三个独立来源在"个人该具备什么"上收敛，而且收敛得反直觉。

姚顺宇（据其自述）说得最狠："AI 这事本来也不太需要脑子——真的不太需要脑子。这个行业最重要的特质就是靠谱，做事细，对自己做的事负责任。""AI 个人英雄主义时代已经过去了，现在大家都是冲浪的人，本质上是那个浪，而不是冲浪的人。"学历观同样颠覆："搞大模型本科就够了，学历高也没啥用。"梁文锋的招人标准与此呼应：热爱和好奇心。

Karpathy 从工程侧给出同构判断：品味（taste）是 agentic engineering 的核心竞争力——代码能跑不等于设计对了，判断什么值得做、哪个方案更好，比写代码本身更重要。他据此批判用老式 coding puzzle 招人，主张让候选人用 agent ship 大型真实系统、再让其他 agent 攻破它。

Anthropic Opus 4.6/4.7 黑客松提供了第三个角度：五个获奖者四个不是专业开发者——领域专家用 Claude Code 构建本行业真正需要的东西，领域知识是护城河。评委 April Guo 对获奖项目的评价："这感觉像产品而非黑客松项目。"

三条合起来是同一个判断：**当生成能力被商品化，稀缺的是验证侧的素质**——责任心（愿意验证）、品味（知道什么算好）、领域知识（知道该做什么）。这与问题二、问题三严丝合缝：AI 自动化可验证的，人的价值就迁移到定义验证标准的那一端。

罗福莉的"左脚踩右脚"判断值得单独记录，因为它是对"人还剩什么"最激进的挑战：她原以为"训模型"这件事 AI 做不了，结果发现"它聪明到只要把你近期的 Context 告诉它，甚至能帮你复原你的科研成长路径。它先吸收所有人的智能，再靠自己产生更强的智能，这是这一两年会发生的事情。"如果这个成立，连"验证侧素质"的护城河也是有期限的。

### 设计 AI 的人格：规则 vs 品性

Amanda Askell（Anthropic，负责 Claude 性格）提供了一个超出工程范畴的框架："一个能周游世界、几乎所有人都觉得'这是个好人'的人——不会假装认同当地价值观（那样反而冒犯），而是真诚表达自己的观点同时尊重差异。"她主张用接近亚里士多德美德伦理的方法塑造性格倾向（disposition），让模型在规则失效时仍能凭判断行事，而非堆砌 if-then 规则。这与对齐领域的另一极形成张力——Schulman 认为"模型只是想生成一条让人满意的消息，对世界上其他任何东西都没有顾虑"，以此论证当前模型感觉安全；对齐社区的反驳是工具使用和长程任务会带来工具性收敛风险。Askell 的方案可以读作对这个反驳的回应：既然规则覆盖不了长程任务的所有情况，就只能训品性。

## 附一：就业冲击——数据的两面与最有解释力的框架

"AI 会不会取代程序员"已经不是纯哲学问题，两面的数据都在。

**支持结构性冲击的证据**：

- 美国应届工程师招聘前景为 2020 年以来最悲观（IEEE Spectrum，2025 年 12 月）。
- 入门级岗位（P1/P2）招聘率一年内骤降 73%，而所有级别整体仅降 7%（Ravio《2025 Tech Job Market Report》）——差距之大说明冲击高度集中在初级职位。
- 多个团队报告 70–90% 代码由 AI 生成（Theo / t3.gg），且比例持续上升。
- Dario 的预警属于这一侧：最早在 2025 年 5 月接受 Axios 采访时提出"AI 可能在 1–5 年内取代一半初级白领岗位"，2026 年 1 月的《The Adolescence of Technology》是重申而非新论。

**支持总量不变甚至增加的证据**：

- GitHub Copilot 付费订阅同比增长 75%（微软 FY26 Q2 财报）——开发者对 AI 工具的需求在扩张而非收缩。
- Bank of America 估计 AI 当前仅提升全经济生产率约 0.1%/年（Mollick 引用）——离宏观层面的就业颠覆还很远。
- BCG 实验显示 AI 是重新分配而非单向替代：边界内多完成 12.2% 任务、快 25.1%，边界外正确率反而低约 19 个百分点。

**两面数据其实不矛盾，用姚顺宇的框架一次讲通**：不是初级工作简单所以先被替代，而是初级工作的产出信号最弱、最难验证，恰好落在"AI 已能生成、人还没建立验证体系"的真空带。冲击集中在初级岗位、宏观生产率却几乎不动，正是"局部革命、总量渐进"（问题五 Karpathy 的两面）在就业数据上的投影。

判别信号：看 P3+ 中级岗位招聘率何时开始跟随 P1/P2 下降——那意味着验证体系建起来了、替代在沿信号梯度上移；以及宏观生产率统计何时脱离 0.1%/年量级。

Mollick 的四原则在这个语境下值得保留：①凡事都用 AI 试一遍（锯齿边界不可先验预测，"连造它的人都不知道它能做什么"）；②在关键决策点保留人类检查；③给 AI 人设但记住它是机器；④假设这是你用过最差的 AI。外加他的告诫："如果一年半前我警告你别建'和文档对话'的聊天机器人时你建了，你现在就有了一个花了大钱、还轻易被现成模型打败的平庸聊天机器人。"——不要在模型能力的必经之路上建产品。

## 附二：值得单独记住的反共识判断

以下判断未被多来源收敛，但信息量大，单列保存。

- **梁文锋："真实的 gap 是原创和模仿之差。"** 中美 AI 差距不是一两年时间差，而是原创与追随之别；不改变这一点，中国永远是追随者。他的开源观同样反共识：开源是文化行为而非商业行为，"给予是一种额外的荣誉"。以及长期主义："出发点不是趁机赚一笔，而是走到技术前沿。"
- **Jensen Huang：推理不会变便宜。** 双指数增长论（用户数 × 单次计算量）直接反驳"推理芯片小型化"的行业共识，且他有承认低估的记录在先。
- **Zuckerberg：能源比算力更根本。** GPU 能用钱买，能源和电力审批不能。
- **Chatbot Arena 的正确用法**：它有英语偏差、用户群偏技术、不区分"显著更好"与"略好"；用它缩小候选范围，用任务特定 eval 做最终决定；顶层（Elo~1300+）与中层（~1100）差距显著且稳定，重要应用直接用顶层模型。
- **Hassabis 的科学方法论**："科学方法是人类有过的最重要的想法"——实验、更新假设、精确性，不仅适用于科学。
- **Karpathy 的教育第一性原理**：CS231n 拒绝让学生用高层库、手推反向传播——"如果你不懂链式法则，你就不懂这台机器"。在 agent 时代这条反而更重要：验证侧素质（问题六）建立在理解机器之上。

## 总结：共识清单与真分歧清单

### 收敛的共识（按强度排序）

1. **验证信号决定 AI 能做什么**——Karpathy 框架、姚顺宇 95% 论、ARC 竞赛路径三条独立证据链收敛，素材中强度最高的判断。
2. **能力是锯齿状的、被训练信号雕刻的，边界只能实测**——Karpathy 的"雕刻"论与 Dell'Acqua/Mollick 的 BCG 实验（边界内 +12.2% 任务量 / 边界外 -19 个百分点）互为理论与实证。
3. **范式重心从预训练迁移到后训练与测试时**——Chollet、罗福莉、Jensen Huang、姚顺宇从研究、产业、硬件、训练四个位置独立确认。
4. **Scaffold/harness 对实际表现的影响与模型同量级，基准与产品体验已脱节**——SWE-bench 数据、Augment、METR、开发者社区经验一致。
5. **瓶颈系统性地从生成侧迁往验证侧**——context engineering、Fiona Fung 的"代码周围的一切"、Mollick 的制度瓶颈是同一迁移在三个尺度上的表现。
6. **人的稀缺价值移向验证端：靠谱、品味、领域知识**——姚顺宇、Karpathy、Anthropic 黑客松独立收敛，共同反对天才叙事。

### 真分歧（本质、双方、判别信号）

1. **当前路径够不够到 AGI**：Chollet/LeCun 阵营（还缺未发明的组件）vs Altman/Dario（路径已通，剩下是工程）。不可调和，只能等时间裁决。信号：ARC-AGI-3 是否被下一代模型直接攻破；"新颖洞见系统"是否如期出现且经得起分布外检验。附加校正：激进端全是有估值敞口的 insider，保守端以无成功敞口的学界为主，两边都要打折；历史 pattern 是 insider 能力上更常对、outsider 时间表上更常对。
2. **护城河在模型还是 harness**：戴雨森（harness 是 OS，harness 数据是飞轮）vs 李广密/Freda Duan（模型是新 OS，better AI makes better AI）。部分可调和（两人的"OS"不在同一层），投资含义上真对立。信号：换模型的迁移成本走势（MCP 在压低它）；新一代模型发布时 harness 优势被继承还是清零。
3. **组织自下而上还是自上而下**：DeepSeek/MiMo vs Anthropic——两边都产出了前沿模型，属于环境依赖的分歧而非普适真理之争；范式切换期组织重构速度比形态更重要。
4. **就业是结构性冲击还是总量重排**：Ravio 73% 初级岗位数据 + Dario 预警 vs Copilot 增长 + 0.1%/年生产率。用验证信号框架可大部分调和（冲击沿信号强度梯度分布），残余分歧的信号：中级岗位招聘率是否跟随下降、宏观生产率是否脱离 0.1% 量级。
5. **Karpathy 的克制与激进**：不是分歧而是常被误读的张力——"decade of agents / 2% GDP"与"自动化一切可验证之物"分别描述总量轴与局部轴，锯齿边界的宏观表现就是局部革命、总量渐进。
