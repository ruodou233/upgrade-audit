# web-research-standards — 联网信息质量共享规范

> 适用：所有联网搜索/信息整合类 skill 与 automation。思路源自 last30days-skill（github.com/mvanhorn/last30days-skill），2026-07-07 引入。版本：2026-07-07。
> 定位：核心思路，不是操作手册；与具体 skill 既有流程冲突时，以更能提高信息质量的一方为准。

## A 先把查询问对（搜索前）

差查询搜不出好结果。开搜前自查：词是否过泛、实体是否同名歧义、数字/型号是否会招来无关内容、"怎么用 X"式措辞是否该改成讨论区说法、非英文主题是否选了真正索引该语言的平台。有问题就花一回合澄清或改写，别硬搜。非交互场景不提问，按最合理解释执行并在产出中标注假设。

## B 评论区与信息质量（采集与综合时）

1. 评论区信号重于帖子信号：帖子热度只说明话题热；最高赞评论（尤其是反驳性的）是重要校正信号，作为共识候选或反证线索去核查，而非直接当共识——评论区也受圈层、算法、站队影响。
2. 关键观点尽量带署名原话引用（u/xxx、@xxx），少转述——转述最容易掺水。
3. 跨源独立印证是质量信号：同一判断在多个互不相干的平台反复出现，可信度升；孤例降。
4. 一手发言优先：当事人自己的帖子/回应优先于第三方转述。
5. 证据权重按问题类型分：判断采用趋势、真实使用、市场预期时，用钱和行为投票的信号（市场赔率、提交记录、迁移）优先，注意赔率流动性与行为规模；判断规格、价格、政策、API、法律等可核验事实时，以一手官方/原始文档为准。厂商营销材料始终降权。

## C 交付时说清覆盖面

产出开头交代：实际用了/合并/跳过哪些源，哪里证据弱或没搜到；别把残缺覆盖铺成"看起来很全"。某个源失败就明说（错误信息脱敏 token/密钥/Cookie 等敏感内容），不悄悄换源。

## 消费者

- domain-explorer（开源镜像）：SKILL.md 内联极简版 + 锚点注释 `<!-- web-research-standards:<模块>@<版本日期> -->`，对外自包含不引用本文件；升级审计核对锚点版本与本文件一致。
- free-token-eggs（开源镜像）：仅内联 C（交付覆盖面）极简版 + 锚点。
- polymarket-anomaly-watch（scheduled task）：接入 `web-research-standards:C@2026-07-07`；任务 prompt 内联 C 极简版，不复制细则。
- 后续联网搜索/信息整合类 skill 与 automation 在此登记；内部消费者直接引用本文件。定时监控类规范由各 automation 方案自定义。
