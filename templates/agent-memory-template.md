# 分层记忆主文档模板

> 用途：作为 Agent 每次会话可读取的长期记忆入口。这里只放跨任务、跨项目、长期稳定的信息；场景化手册、长背景、排障细节和具体工作流应放到 reference 或 skill。
>
> 使用方法：复制本模板为你自己的记忆主文档，然后删除示例，填入真实内容。以下示例均为虚构，只展示写法，不代表作者偏好。
>
> Skip 记录与暂存信号见配套 `templates/audit-triage-template.md`；首次使用时复制为你自己的 `audit_triage_file`（默认 `references/audit-triage.md`），不放在本记忆主文档——这类内容只服务审计类工作，不需要每次对话都加载。

## 管理原则

**用途**：记录维护这份记忆文档的原则，帮助 Agent 判断什么该写入、什么该删除、什么该下沉到其他文档。

- 示例：只记录跨项目长期适用的信息；只影响一次任务的细节不进入本文件。
- 示例：每次新增规则时，同时检查是否已有相近规则，避免同义重复。

---

## 用户偏好

**用途**：记录使用者长期稳定的协作偏好、输出偏好和默认工作方式。偏好应可执行、可验证，避免写成性格评价。

- 示例：讨论方案时先给两个可选路径，再说明推荐路径和取舍。
- 示例：交付表格类结果时，默认同时给可复制的 Markdown 表和一句摘要。

---

## 活跃索引

**用途**：记录当前仍在使用的 skill、自动化、长期项目和关键文档入口。这里只放索引和状态，不展开完整操作手册。

| 名称 | 用途 | 状态 |
|---|---|---|
| receipt-parser | 解析票据截图并整理为报销表 | 活跃；详细流程见对应 skill |
| weekly-brief | 汇总一周项目进展 | 试用中；需要继续观察输出稳定性 |

---

## 架构决策

**用途**：记录仍然有效、会影响后续任务的结构性决策。写清决策、原因、适用范围和何时重新评估。

- 示例：项目素材统一放在 `assets/` 下，避免散落到各页面目录；如果构建工具改版，再重新评估。
- 示例：自动化脚本默认先输出 dry-run 报告，再由使用者确认是否执行写入。
